# 0.1.0 — Initial release

The first public release of Grove on npm as `grovemd`, with the
`grove` CLI. Grove is a local markdown wiki: point it at any folder
and get a browseable Angular SPA with live rendering, math,
diagrams, syntax highlighting, and media previews. No database,
no cloud, no auth — a single-user localhost tool.

## Added

### Local markdown wiki

- **Browse any folder** — `grove <folder>` (or `npx grovemd <folder>`)
  starts an Express server with an Angular SPA that renders the
  folder's contents as a wiki. Directory tree in the sidebar,
  markdown rendering in the main pane.
- **GitHub-flavored markdown** — tables, task lists, strikethrough,
  footnotes, fenced code blocks.
- **Math** via KaTeX — `$inline$` and `$$block$$` expressions.
- **Diagrams** via Mermaid — flowcharts, sequence diagrams,
  and all other Mermaid-supported types.
- **Syntax highlighting** via highlight.js — 190+ languages
  auto-detected or declared on fenced-code fences.
- **Media previews** — images, video, audio, and PDF render inline
  from their file path.
- **Internal links** — relative markdown links route through the
  Angular router without a full page reload; GFM-style heading
  anchors support fragment scrolling.

### Static wiki bundle

- **`grove build-wiki`** subcommand emits a self-contained
  GitHub-Pages-ready SPA bundle to `dist/frontend/wiki/`. Same
  renderer as the server, but with documents baked into a JSON
  manifest — no server needed. See
  [Wiki mode](../architecture/wiki-mode.md) and
  [Wiki deployment](../guides/wiki-deployment.md).

### External-tool integration

- **Capability-gated buttons** for Terminal, Zed, and
  Claude Code. `GET /api/capabilities` reports which tools the
  host supports; the UI hides buttons for unsupported tools. Only
  Terminal and Claude Code are advertised as supported on macOS;
  Linux and Windows get a platform-appropriate subset.

### HTTP surface

- `GET /api/documents?path=` — directory listing, dotfiles hidden.
- `GET /api/documents/raw?path=` — raw file bytes for rendering.
- `POST /api/open` — launches the external tool on the host
  (zod-validated body).
- `GET /api/capabilities` — platform and supported-tool report.
- `/_content/<path>` — `express.static` mount for binary media
  (images, video, PDF, etc.).
- `/{*splat}` — SPA catch-all.

## Security

- **Path containment.** `ensureInside(docsDir, userPath)` in
  `server/path-sandbox.ts` uses `realpath` to close symlink-escape
  and NUL-byte bypasses. Every user path is resolved before any
  fs call.
- **Localhost-only posture.** No cookies, no sessions, no auth —
  Grove assumes it's running on the user's machine. The threat
  model is "other browser tabs", not "the internet".

## Known limitations

- No write surface yet; Grove is read-only in 0.1.0. The editor
  arrives in [0.3.0](./0.3.0.md) behind `--allow-edits`.
- No search UI. Use the browser's in-page find, or a terminal
  tool like `ripgrep` against the folder.
