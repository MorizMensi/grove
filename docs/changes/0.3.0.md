# 0.3.0 — 2026-04-21

The "Grove grows an editor" release. Grove is still a read-only
wiki by default, but with two opt-in CLI flags it turns into a
local-first, Typora-style markdown editor with a full CRUD surface
over the docs folder and optional git auto-commit.

Also in this line: a `--disable-security` escape hatch for
consumers who need specific safety checks relaxed, plus several
editor UX fixes and the HTML preview theme passthrough fix.

## Added

### In-browser markdown editor

The editor is surfaced as a pencil toggle on every `.md` file.
It's gated behind two opt-in CLI flags so the default posture
remains read-only:

- **`--allow-edits`** unlocks the write surface.
- **`--git-commit`** (additionally) commits each write.

#### Backend — write surface

- **CRUD routes on `/api/documents`:**
  - `GET /api/documents/raw?path=` — raw file with `mtime` and
    `ETag`.
  - `PUT /api/documents?path=` — write with `If-Unmodified-Since`
    precondition and atomic `tmp`-then-`rename`. Returns
    `409 mtime-conflict` on stale writes.
  - `POST /api/documents?path=` — create file (or directory with
    `?kind=dir`). Returns `409 parent-missing` when the parent
    doesn't exist.
  - `DELETE /api/documents?path=` — delete file or empty
    directory.
- **Gates.** `requireEdits` (the `--allow-edits` check) runs
  before `csrfOrigin` (Origin-equals-Host) which runs before
  `Content-Type: application/json` which runs before the 10 MB
  JSON limit. Name validation rejects traversal, NUL bytes, and
  platform-reserved names.
- **Path containment.** `server/path-sandbox.ts` consolidates
  `realpath`-hardened containment for every path-consuming
  handler — the one place to audit.
- **Git auto-commit.** `--git-commit` produces one
  `grove: <verb> <rel>` commit per write via
  `execFile('git', [...])`. Validated at startup (repo must
  exist, user.name/user.email must be set); "nothing to commit"
  is swallowed rather than erroring.
- **Zed integration was removed** entirely to free the pencil
  slot on the document shell toolbar. `'zed'` is no longer a
  valid `OpenAction`; `POST /api/open` with `action: 'zed'`
  returns 400 with the zod error format. `ZED_BIN` is no longer
  read; `supports.zed` is no longer exposed; `server/zed-resolver.ts`
  was deleted.

#### Frontend — the editor itself

- **CodeMirror 6 host.** `features/editor/editor.component.ts`
  hosts an `EditorView` with Grove's theme bindings, a
  `SaveService`, and a `canDeactivate` guard.
- **Typora-style inline reveal.** `hybrid-markdown.ts` runs a
  `StateField<DecorationSet>` that hides inline markdown syntax
  (`**`, `_`, `` ` ``, link brackets, heading `#`) when the
  caret is outside the construct and reveals the raw syntax
  when inside it. Visually identical to view mode until you
  start editing.
- **Live block widgets.** `DlBlockWidget` replaces fenced code,
  tables, images, and Mermaid diagrams with real
  `DlNodeComponent` mounts — the same renderer as view mode —
  and collapses back to raw source when the caret enters the
  block. Math blocks stay as raw text in v1.
- **Save flow.** `SaveService` binds `⌘S`, tracks `mtime`, and
  on `409` surfaces a **Reload / Overwrite / Cancel** banner.
  `canDeactivate` plus `beforeunload` protect dirty buffers
  from navigation and tab close.

#### Frontend — shell and accessibility

- **Pencil gating.** Shown only when
  `capabilities.supports.edits` is true. The `auto-commit` pill
  next to the pencil appears when `supports.gitCommit` is true.
- **Sidebar CRUD.** Right-click (or **Shift+F10**) opens a
  context menu with **New file**, **New folder**, **Delete**.
  Directory rows get an inline `+` affordance. Delete goes
  through a confirm modal with a focus trap; deleting the
  currently open file navigates to the parent and announces
  through a singleton `LiveRegionComponent`.
- **Keyboard.** `Alt+N` creates a new file (browsers reserve
  `Cmd+N` for a new window); `F2` announces "Rename is not
  available yet" — placeholder for a future release.
- **A11y.** Menu, dialog, and toggle patterns follow the
  WAI-ARIA Authoring Practices Guide; animations respect
  `prefers-reduced-motion`.

### `--disable-security <csv>` CLI flag

A comma-separated list of named escape hatches for the server's
built-in safety checks. May be repeated (multiple `--disable-security`
invocations merge). Prints a stderr warning on startup whenever any
feature is disabled. Unknown values fail startup with a listing of
the valid flags.

Initial value:

- **`allow-symlinks`** — lets symlinks inside the docs folder
  resolve to targets *outside* it on the read, write, and open
  routes. The *addressed* path must still sit inside the docs
  folder, so `..` traversal and sibling-directory prefix bypass
  remain blocked. Intended for users whose notes link out to a
  shared assets tree.

See [CLI reference](../reference/cli.md) for the full flag list.

## Fixed

### HTML preview theme passthrough

The CSP sent with `/_content/*.html` and `/_content/*.svg` now
includes `sandbox allow-same-origin`. Without that token the
browser forced the response into an opaque origin, which silently
blocked the parent page from reading `iframe.contentDocument` and
injecting theme CSS variables — previews rendered with unthemed
colors. Scripts remain blocked by `script-src 'none'` and by the
absence of `allow-scripts` in both the iframe `sandbox` attribute
and the CSP sandbox directive.

### Editor: click below block widget lands on correct line

Clicking a line below a rendered block widget (Mermaid, fenced
code, table, block image) now places the caret on that exact line
instead of 5–7 lines further down.

Root cause: `DlNodeComponent`'s `:host { display: contents }`
applied to the block widget's host element, giving the host a
0×0 box even while its rendered Mermaid SVG occupied ~140 px.
CodeMirror's heightmap measured 0 and fell back to the widget's
`estimatedHeight` (48 px), so every line below the widget was
located (realHeight − estimatedHeight) / line-height ≈ 5–7 lines
earlier in the height map than in the actual DOM, and
`posAtCoords` resolved clicks accordingly.

Fix: scope a `:host(.cm-dl-widget) { display: block }` override
in `dl-node.component.scss` so the host becomes a real layout box
inside the editor. Additional hardening: widget `mousedown`
branches on top/bottom half (top reveals source, bottom hops past
the widget), `ResizeObserver` calls `view.requestMeasure()` on
host resize, and the widget's vertical rhythm uses `padding`
rather than `margin`.

### Editor: caret enters inline spans, Backspace is per-character

Caret can now be placed inside bold, italic, inline-code, and
link spans, and Backspace/Delete removes one character instead of
the entire span. The hybrid-markdown field was contributing its
whole decoration set to `EditorView.atomicRanges`, so the outer
`mark` ranges (styling the full `**bold**` / `` `code` `` / etc.)
were treated as atomic. Only the `hide` (replace) decorations
over `**`, `_`, `` ` ``, and link brackets are now atomic, so
arrow keys still skip hidden markers as before.

### Editor: pencil appears on `.md` files again

`DocumentShellComponent.isEditableFile` read `mode` as a plain
field rather than as a signal, so its memo never invalidated
when mode transitioned `loading → file` — the pencil stayed
hidden whenever the capabilities fetch won the race against the
first document load.

### Editor: save/reload/delete use correct filename

Edit, save, reload-after-conflict, overwrite-after-conflict, and
delete now pass the actual filename (stem + extension) to the
server. Previously the shell sent only the URL stem for `.md`
files (e.g. `path=how-it-works`), so `/api/documents/raw`
404-ed, PUT would have silently written to an extensionless
sibling, and sidebar delete on a listing entry returned 403.

### Mobile document shell

<768px: swapped the two-column grid for a single-column flex
layout. Sidebar now starts closed and dismisses on backdrop or
link tap (with proper `aria-label` / `aria-expanded`); content
fills the viewport; long pages scroll instead of being clipped.

## Changed

### Action-button bar and confirm-dialog styling

Unified the action-button bar at the top of the document shell so
every toggle shares one hover and one active treatment: hover gives a
real background change (not just a color shift), and the active state
is a single pill-shaped chip — no more mix of bold-colored-text for
the pencil and squared colored background for the preview/source
radios. The "Edit" label is removed from the pencil (icon only when
idle); "Done" still shows next to the filled pencil, centered with the
icon. The "Unsaved changes" confirm dialog (Save / Discard / Cancel)
is restyled to use Grove's design tokens (surface, border, shadow,
spacing, focus ring) instead of hardcoded fallbacks.

## Removed

- **Zed integration.** Removed entirely. Zed's capability-gated
  button on the document shell toolbar was replaced with the
  editor pencil. The `'zed'` action, the `ZED_BIN` env var, and
  `server/zed-resolver.ts` were all deleted. `POST /api/open`
  with `action: 'zed'` now returns 400 with the zod error
  format. Any lingering frontend reference to `'zed'` is a bug.
