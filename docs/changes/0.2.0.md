# 0.2.0 — 2026-04-20

The "harden the preview surface" release. Grove grew two new
previewable file types (HTML and SVG), and the work of making them
safe drove a series of `/_content/` hardening fixes that apply to
every consumer's docs folder — not just HTML.

## Added

### HTML and SVG file previews

- **`.html` files** now render inside a sandboxed iframe with a
  **Preview / Source** toggle above. The iframe inherits Grove's
  active theme through CSS-custom-property injection, so an
  HTML file previewed under the Saffron dark theme picks up
  Saffron's colors without the author doing anything.
- **`.svg` files** preview as an inline image, with a
  source-view fallback that shows the SVG as fenced XML in a
  syntax-highlighted block.
- See [file types](../reference/file-types.md) for the full
  preview matrix.

## Security

HTML previews necessarily execute markup under the same origin as
Grove itself; the surface was hardened to make that safe.

### Content-Security-Policy on `/_content/`

- HTML and SVG responses under `/_content/` now ship with:
  ```
  Content-Security-Policy: sandbox; script-src 'none'; object-src 'none'; base-uri 'none'
  X-Content-Type-Options: nosniff
  ```
- The `sandbox` directive is the backstop; `script-src 'none'`
  blocks scripts even if a consumer's HTML tries an inline
  `<script>`; `object-src 'none'` blocks `<object>`/`<embed>`;
  `base-uri 'none'` stops a malicious file from rewriting
  relative URL resolution.

### Iframe sandbox invariant

- The iframe used for HTML preview sets
  `sandbox="allow-same-origin"` and **never** `allow-scripts`.
  This is a load-bearing invariant: flipping it would turn every
  HTML file in a consumer's docs folder into an XSS vector on
  `localhost`.
- Enforced at **prepublish** by
  `scripts/check-sandbox-invariant.mjs` — a static grep that
  fails the publish if `allow-scripts` appears in any iframe
  `sandbox` attribute. Replaces an orphan Jasmine spec that was
  never wired to a test runner.

### Dotfile deny on `/_content/`

- The static mount at `/_content/` refuses dotfiles: `GET
  /_content/.env` returns 403, not the file or the SPA
  `index.html`. Configured with `dotfiles: 'deny'` and
  `fallthrough: false` so the 403 isn't masked by the Angular
  catch-all.
- **Trade-off.** `.well-known/` and similar directories under a
  consumer's docs folder are no longer served. Grove is a wiki,
  not a web root — this was judged the right default.

### Sibling-directory prefix bypass closed

- Path containment in the documents and open routers now
  requires an explicit path-separator boundary. Previously a
  path like `/foobar` could satisfy "starts with `/foo`"; now
  the check demands `/foo` + separator, so `/foo` and `/foobar`
  are correctly treated as disjoint trees.

## Changed

- **DOMPurify removed** as a runtime dependency. HTML previews
  now rely on iframe sandboxing for XSS isolation; scrubbing
  markup before rendering was both slower and weaker than a
  real sandbox. Bundle size dropped accordingly.
- **`prepublishOnly`** now runs `check:sandbox` *before*
  `build:all`, so a sandbox regression blocks the publish before
  the long build step.

## Known issues

- **`@angular/core ^19.2.0`** carries
  [GHSA-g93w-mfhg-p222](https://github.com/advisories/GHSA-g93w-mfhg-p222)
  (XSS via i18n attribute bindings). Grove does not use Angular
  i18n attribute bindings on user-controlled data; the patch
  requires an Angular 19.3+ bump, deferred.
